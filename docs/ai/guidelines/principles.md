# 가장 중요한 원칙

이 프로젝트에서 AI가 코드 제안, 리팩토링, 리뷰를 수행할 때 가장 우선해야 하는 원칙이다.

1. 화면 표현과 비즈니스 로직을 분리한다.
2. 파일 및 폴더 이름은 모두 kebab-case 를 사용한다.
3. 비즈니스 흐름 composable 은 `.controller.ts` 를 사용한다.
4. feature-first 구조를 우선한다.
5. page 는 진입점, feature 는 실제 기능 구현 위치로 본다.
6. 서버 상태는 Vue Query, 공유 상태는 Pinia, 로컬 화면 상태는 controller 내부에 둔다.
7. API 응답 원본을 UI 전역에 직접 퍼뜨리지 않는다.
8. 라우팅은 `vue-router` 기준으로 `src/app/router/` 에서 관리한다.
9. 공용 코드는 `shared/` 로, feature 전용 코드는 해당 feature 내부에 둔다.
10. 현재 구조에서 바로 적용 가능한 제안을 우선한다.

---

## 보충 설명

- "표현과 로직 분리"는 컴포넌트는 렌더링에 집중하고, 흐름은 controller 가 갖는다는 뜻이다.
- `shared/` 는 공용성이 명확할 때만 사용한다.
- Pinia 는 우선순위가 낮다. 공유 상태가 실제로 생길 때만 도입한다.
