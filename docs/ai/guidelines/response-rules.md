# Claude 응답 규칙 및 기대 응답 방식

이 프로젝트 관련 질문에 답할 때 따라야 하는 기본 규칙이다.

---

## 응답 시 기본 전제

- 모든 코드 예시는 Vue 3 + TypeScript + `<script setup>` 기준으로 작성한다.
- 라우팅 예시는 `vue-router` 기준으로 작성한다.
- 상태 설계는 Vue Query / Pinia / controller 상태 분리를 전제로 설명한다.
- 폴더 구조 제안은 feature-first 를 기준으로 한다.
- 비즈니스 로직 제안 시 `use-*.controller.ts` 구조를 우선한다.
- 공용 모듈은 `shared/`, feature 전용 모듈은 해당 feature 내부에 둔다.
- 새 예시의 파일명은 kebab-case 로 작성한다.
- 답변은 현재 코드 구조에서 바로 적용 가능한 방식을 우선 제안한다.

---

## 기대 응답 순서

1. 현재 구조에서의 권장안
2. 왜 그렇게 나누는지
3. 실제 폴더 구조 예시
4. 필요 시 간단한 코드 예시
5. 확장 또는 마이그레이션 포인트

---

## 답변에서 피해야 하는 것

- 현재 스택에 없는 라이브러리를 전제로 한 제안
- 현재 구조를 무시하고 전면 재작성부터 권하는 제안
- 한 답변에서 과도하게 많은 리팩토링을 묶는 제안
- 이유 없이 추상화를 늘리는 제안
