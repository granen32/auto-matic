# D3 작성 규칙

D3는 반드시 분리한다. Vue 컴포넌트 안에 긴 D3 selection 체인을 작성하지 않는다.

---

## 권장 흐름

1. API 또는 store에서 데이터 조회
2. controller에서 차트용 데이터로 가공
3. component에서 `ref`와 lifecycle 관리
4. renderer에서 draw / update / cleanup 수행

---

## 권장 파일 구조

```txt
features/
  dashboard/
    components/
      sales-chart.vue
    controllers/
      use-sales-chart.controller.ts
    lib/
      sales-chart-renderer.ts
      sales-chart-options.ts
    types/
      sales-chart.types.ts
```

---

## 책임 분리

| 파일 | 책임 |
| --- | --- |
| `sales-chart.vue` | DOM ref 보유, mount/unmount 시 renderer 호출, props 전달 |
| `use-sales-chart.controller.ts` | API 데이터 → 차트용 데이터 가공, 필터 상태 관리 |
| `sales-chart-renderer.ts` | D3 selection, scale, axis, transition, draw / update / cleanup |
| `sales-chart.types.ts` | 차트용 데이터 / 옵션 타입 |

---

## 컴포넌트 예시

```vue
<script setup lang="ts">
import { onMounted, onBeforeUnmount, ref, watch } from 'vue'
import { createSalesChartRenderer } from '@/features/dashboard/lib/sales-chart-renderer'
import { useSalesChartController } from '@/features/dashboard/controllers/use-sales-chart.controller'

const root = ref<HTMLDivElement | null>(null)
const { chartData } = useSalesChartController()
let renderer: ReturnType<typeof createSalesChartRenderer> | null = null

onMounted(() => {
  if (!root.value) return
  renderer = createSalesChartRenderer(root.value)
  renderer.draw(chartData.value)
})

watch(chartData, (next) => renderer?.update(next))

onBeforeUnmount(() => renderer?.destroy())
</script>

<template>
  <div ref="root" class="w-full h-[320px]" />
</template>
```

## 규칙 요약

- Vue SFC 안에 D3 selection 체인을 길게 두지 않는다.
- D3는 렌더링, 스케일, axis, transition에 집중한다.
- Vue는 props, ref, watch, lifecycle을 담당한다.
- 데이터 가공은 controller에서 끝낸 뒤 renderer에 전달한다.
- renderer는 외부에 `draw / update / destroy` 같은 명시적 API만 노출한다.
