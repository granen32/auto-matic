# D3 작성 규칙

현재 프로젝트에는 D3 의존성이 없다.
따라서 이 문서는 **도입 시 적용할 확장 기준**으로 본다.

---

## 도입 원칙

- D3 를 도입하더라도 Vue 컴포넌트 안에 긴 selection 체인을 직접 작성하지 않는다.
- 차트 전용 feature 내부에 `components / controllers / lib / types` 를 분리한다.
- renderer 는 `lib/` 에 두고, component 는 DOM ref 와 lifecycle 만 담당한다.

---

## 권장 구조 예시

```txt
features/
  analytics/
    components/
      sales-chart.vue
    controllers/
      use-sales-chart.controller.ts
    lib/
      sales-chart-renderer.ts
    types/
      sales-chart.types.ts
```

---

## 메모

- 현재 코드 생성이나 리뷰에서는 D3 를 기본 전제로 가정하지 않는다.
- 사용자가 D3 도입을 명시적으로 요청할 때만 이 문서를 적용한다.
